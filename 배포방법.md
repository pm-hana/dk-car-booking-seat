# 🚗 DK CAR BOOKING — 인터넷 공개(배포) 방법

이 앱을 `https://<앱이름>.streamlit.app` 주소로 만들어, **어디서든 도메인만 입력하면 접속**되게 하는 방법입니다.
예약 데이터는 **Firestore**에 저장되어 서버가 재시작돼도 사라지지 않습니다.

전체 순서: **① Firestore 준비 → ② GitHub에 코드 올리기 → ③ Streamlit Cloud에서 공개**

---

## ① Firestore(데이터베이스) 준비  — 약 5분

1. https://console.firebase.google.com 접속 → 프로젝트 **`dk-car-booking-seat`** 선택
2. 왼쪽 메뉴 **빌드 > Firestore Database** → **데이터베이스 만들기**
   - 위치: `asia-northeast3`(서울) 권장
   - 모드: **프로덕션 모드**로 시작 (아래 3번에서 규칙만 잠깐 열어줌)
3. **서비스 계정 키 발급** (앱이 Firestore에 접속할 열쇠)
   - ⚙️(설정) > **프로젝트 설정** > **서비스 계정** 탭
   - **새 비공개 키 생성** 클릭 → JSON 파일이 다운로드됨 → 이 파일을 잘 보관 (③에서 사용)

> 이 JSON 파일은 비밀번호와 같습니다. **절대 GitHub에 올리거나 남에게 공유하지 마세요.**

---

## ② GitHub에 코드 올리기  — 약 5분

GitHub 계정이 없으면 https://github.com 에서 무료 가입하세요.

1. GitHub에서 새 저장소(repository) 생성 — 이름 예: `dk-car-booking` (Private 권장)
2. 이 폴더에서 아래 명령을 **PowerShell**에 한 줄씩 입력:

```powershell
cd "c:\Users\hana\OneDrive\바탕 화면\DK CAR BOOKING SEAT"
git init
git add .
git commit -m "DK Car Booking 배포 준비"
git branch -M main
git remote add origin https://github.com/<본인아이디>/dk-car-booking.git
git push -u origin main
```

> `.gitignore`가 이미 있어서 비밀 키·가상환경·백업 파일은 자동으로 제외됩니다. 안심하세요.

---

## ③ Streamlit Cloud에서 공개  — 약 5분

1. https://share.streamlit.io 접속 → **GitHub 계정으로 로그인**
2. **Create app > Deploy a public app from GitHub** 선택
3. 설정:
   - Repository: 방금 만든 `<본인아이디>/dk-car-booking`
   - Branch: `main`
   - Main file path: `app.py`
   - App URL: 원하는 앱 이름 입력 → 이게 최종 주소가 됩니다 (예: `dk-car-booking.streamlit.app`)
4. **Advanced settings > Secrets** 칸에, ①에서 받은 JSON 파일 내용을 아래 형식으로 붙여넣기:

```toml
[firebase]
type = "service_account"
project_id = "dk-car-booking-seat"
private_key_id = "JSON의 private_key_id 값"
private_key = "JSON의 private_key 값 (\n 포함 그대로)"
client_email = "JSON의 client_email 값"
client_id = "JSON의 client_id 값"
auth_uri = "https://accounts.google.com/o/oauth2/auth"
token_uri = "https://oauth2.googleapis.com/token"
auth_provider_x509_cert_url = "https://www.googleapis.com/oauth2/v1/certs"
client_x509_cert_url = "JSON의 client_x509_cert_url 값"
universe_domain = "googleapis.com"
```

   - 참고 템플릿: 이 폴더의 `.streamlit/secrets.toml.example`
5. **Deploy** 클릭 → 1~2분 뒤 앱이 뜹니다.

🎉 완료! 이제 `https://<앱이름>.streamlit.app` 주소로 **회사 직원 누구나 어디서든** 접속할 수 있고,
예약은 Firestore에 안전하게 저장됩니다.

---

## 자주 묻는 질문

- **주소를 바꾸고 싶어요** → Streamlit Cloud 앱 설정에서 App URL 변경 가능.
- **코드를 수정하면?** → 수정 후 `git add . / git commit / git push` 하면 자동으로 재배포됩니다.
- **로컬(내 PC)에서는 어떻게 되나요?** → 비밀 키가 없으므로 예전처럼 `bookings.json` 파일에 저장되며 그대로 개발/테스트 가능합니다. 배포 서버에서만 Firestore를 씁니다.
- **`.web.app` 주소로 하고 싶어요** → 그건 Firebase Hosting + Cloud Run 방식이며, Google 결제 계정 연결이 필요합니다. 필요하면 그 방법으로도 안내해 드릴게요 (구성 파일은 이미 준비돼 있음).
